# Go: sqlx, sqlc, and goose

> Sunday 1:37 PM

Woke up at around 7:30 AM and then started exploring db related packages in Go. 

First looked into sqlx and noticed that it uses struct tags as a way of mapping, which seems straight forward enough, and it appears to be generally an extension of Go's standard database/sql library. 

Shortly after, I came across sqlc, which seems to have a lot of positive impression as well. Got curious and checked it out. 

From what I understood, it's main advantage is that you get type-safety. Meaning, you get to catch a query error that would have otherwise shown at runtime. In a nutshell, you write the query, then execute sqlc generate command. Assuming it succeeds, then you get to use the generated type-safe code.I was on the fence but leaning more towards sqlc. While sqlx defintely appears to be simpler (not much setup is needed, and no generated code), sqlc, on the otherhand is also not far off in terms of simplicity, as the generated code is code you would have had written yourself had you only been relying on go's standard database/sql library. Lastly, I checked goose, which appears to be for database migrations.

I spent around more or less 3 hours on this, only to end up with sticking with the standard library. Well, at least for now. I'm sure once I'm working on an actual project, these libraries would actually make a difference in terms of team workflow.
